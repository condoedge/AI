## Key Concepts

### Auto-Discovery
The process of automatically generating RAG/Neo4j configuration from Eloquent model definitions, eliminating manual configuration.

### Fallback Chain
Priority order for configuration resolution:
1. `nodeableConfig()` method (explicit)
2. `config/entities.php` (legacy)
3. Auto-discovery (default)

### NodeableConfig
Fluent builder for overriding auto-discovered configuration. Allows selective customization while keeping defaults.

### Discovery Services
Specialized services that extract configuration from different model aspects:
- PropertyDiscoverer
- RelationshipDiscoverer
- ScopeDiscoverer
- AliasGenerator
- EmbedFieldDetector

### Introspection
Using PHP Reflection to analyze model structure at runtime, enabling auto-discovery.

### Convention over Configuration
Laravel principle applied to RAG integration: sensible defaults based on conventions, explicit config only when needed.

## FAQs

### Q: Do I need to configure anything?
**A:** No! Just add the `HasNodeableConfig` trait. Auto-discovery handles the rest.

### Q: Can I override auto-discovered settings?
**A:** Yes, use the `nodeableConfig()` method with `NodeableConfig::discover($this)` to override specific settings.

### Q: Will this break my existing project?
**A:** No. Existing config files take priority. Backward compatibility is maintained.

### Q: What if auto-discovery gets it wrong?
**A:** Use `nodeableConfig()` to override. See [Quick Start - Override Examples](ENTITY-AUTO-DISCOVERY-QUICKSTART.md#override-examples).

### Q: How fast is auto-discovery?
**A:** <100ms first time, <5ms cached. See [Performance](ENTITY-AUTO-DISCOVERY-IMPLEMENTATION.md#performance-benchmarks).

### Q: Can I preview auto-discovered config?
**A:** Yes! Run `php artisan ai:discover YourModel`

### Q: How do I migrate from config files?
**A:** See [Migration Examples](ENTITY-AUTO-DISCOVERY-EXAMPLES.md#migration-examples)
